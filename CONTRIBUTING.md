# Contribuir

Gracias por querer contribuir.

## Flujo recomendado

1. Hacer fork o crear rama desde main.
2. Nombrar la rama con prefijo:
   - feat/
   - fix/
   - docs/
3. Hacer cambios pequenos y enfocados.
4. Probar localmente antes de abrir PR.
5. Abrir Pull Request con descripcion clara.

## Convenciones

- Mantener codigo modular y reutilizable.
- Evitar duplicacion de logica.
- Preferir nombres descriptivos de funciones y variables.
- Evitar mezclar datos, estado y render en una sola funcion.

## Commits

Formato sugerido:

- feat: agrega selector de nivel
- fix: corrige validacion de nombres duplicados
- docs: mejora guia de instalacion
